Starter guide

Where do I even start?

When you are trying to improve IT for a team or business, it can feel like everything needs to be fixed at once. But most good IT systems start with a few simple questions: How do people ask for help? What tools does the team use every day? What happens when someone joins the company?

Start small

IT does not have to begin with a giant system.

You do not need to start with a huge migration, a perfect help desk portal, or a fully automated workflow. For many small teams, the first useful step is making the current work easier to see.

A practical starting point is usually a clear intake path, a simple list of core tools, and a basic understanding of what happens when someone joins the team.

Ticket intake

Start with how people ask IT for help.

This is often called ticket intake. It simply means the path people use when they need IT help. It does not have to start with a complicated ticketing system.

  • A dedicated email inbox for IT requests
  • A dedicated Slack channel
  • A lightweight help desk tool
  • Jira Service Management, including the free edition, if the team is ready for something more structured

The goal is not to build the fanciest portal. The goal is to give people one clear place to ask for help and give IT one clear place to review the work.

Basic tools

Get clear on the basic tool setup.

Most teams need a few core categories covered before they need anything elaborate. The best tool is not always the fanciest one. It is the one the team can actually use consistently.

Productivity and email

Google Workspace or Microsoft 365 for email, documents, calendars, and collaboration.

Async communication

Slack or Microsoft Teams for coordination, announcements, and quick questions.

Project and task tracking

Jira, Asana, Trello, or a similar tool so work does not live only in memory or DMs.

IT requests

A simple intake path so support work is easier to find, prioritize, and finish.

User lifecycle

Look at what happens when someone joins.

User lifecycle means what happens when someone joins, changes roles, or leaves the company. Start with onboarding. The first step is not automation. The first step is understanding the current process.

  • Who knows when someone is starting?
  • Where does that information live?
  • Who creates the accounts?
  • What tools does the person need?
  • What usually gets missed?
  • What causes last-minute scrambling?

Document first

Document before automating.

Automation works best when the process is already clear and repetitive. If the process is different every time, automation will just make the confusion move faster.

Write down what happens today. Then improve the process in small chunks. A simple checklist that people actually use is often more valuable than a complex workflow nobody trusts yet.
