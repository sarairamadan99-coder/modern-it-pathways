Where I can help

Practical cleanup for the IT workflows teams rely on every week.

The goal is not an enterprise transformation project. It is focused, useful help for the operational places where growing teams start to feel friction.

Workflow and access cleanup

  • Employee onboarding and offboarding workflows
  • Access management and role-based access cleanup
  • Google Workspace, Okta, Slack, Jira/JSM, Jamf, and SaaS admin process review
  • SaaS/vendor cleanup and clearer ownership

Process design and documentation

  • Documentation and repeatable IT process design
  • Lightweight automation opportunities
  • Practical security and compliance-minded IT operations
  • Service desk and internal request workflow improvements

Common signs

You might need this if IT work is starting to depend on memory.

New hires rely on too many manual steps to get access
Offboarding depends on memory or scattered checklists
HR, IT, Finance, and managers are not aligned on employee lifecycle steps
SaaS access ownership is unclear
Documentation exists but is outdated or hard to use
The team is growing but IT processes still feel informal
